# Button
ボタンコンポーネントは、ユーザーアクションのトリガーとして使用される重要なUIコンポーネントです。

# Usage
## Appearance
`Appearance`は、画面内での情報のヒエラルキーを表現するために使用します。情報の優先度を踏まえ、強弱がつくように設計してください。すべてを強調すると、すべてが同じに見えます。
- `Fill`
  - CTAをより強調したい場合に使用します。（例：レポートのダウンロード）
  - 複数ボタンが並ぶ箇所での最終実行処理のアクショントリガーとして使用します。
  - 画面に１つ程度の割合で使用します。
- `outline`
    - 基本のボタンです。
    - 役割レベルが並列の場合、ページ内、機能内での複数使用することが可能です。
- `text`
    - 機能に対するアクションなど、サブ的アクションのトリガーとして使用します。
    - 基本ボタンと比べ、役割レベルが低い場合に使用します。
    - 用途をより差別化したい場合にFillとセットで使用する場合もあります。

## Size
**サイズ可変**
- **small**: Table内で使用します。
- **medium**: 基本的に使用されるサイズです。
- **large**: ユーザーに明示的に押させたい時に使用します。
- **x-large**: ログイン画面で使用します(仮)。

**サイズ固定**
**サイズ可変**
- **small**: Table内で使用します。
- **medium**: 基本的に使用されるサイズです。
- **large**: ユーザーに明示的に押させたい時に使用します。
- **x-large**: ログイン画面で使用します(仮)。

**サイズ固定**
- **width-160**: 可変にするとボタンが小さく感じる場合や、レイアウトのズレが起きる時に使用します。
- **width-80**: モーダル内のボタンで使用します。
